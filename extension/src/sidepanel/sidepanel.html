Feature Guide

QA Deck lives in the Chrome extension side panel — click the extension icon in the browser toolbar to open it. Use the tabs below to navigate features.

Scan
Extracts all interactive elements from the current page (forms, buttons, inputs). Required first step before generating tests.
Test Cases
AI-generated test cases from the scanned page. Edit, filter, approve, or add manual cases before generating scripts or packs.
Journey NEW
Build a multi-page flow by adding scanned pages or importing a capture session. QA Deck groups journey-level and step-level coverage so you can generate end-to-end suites with clear transition gaps.
Scripts
Generates ready-to-run automation code in Selenium Python/Java and Playwright Python/TypeScript from approved test cases.
CI/CD
Generates GitHub Actions or Jenkins pipeline configs to run your generated tests automatically in CI.
Locators NEW
Click Start Inspecting, then click any element on the page to instantly see Playwright and Selenium locators. Test any locator with the built-in verifier.
Capture
Local-only feature. Launches a Playwright browser on your machine to record interactions. Requires the backend running locally (node server.js). Not available on the cloud backend.
Test Datasets

Each row becomes a parametrized test run. Column headers are test data keys.

Keys (headers)
Environment Config

Define environments to generate separate config files. Tests read ENV at runtime.
